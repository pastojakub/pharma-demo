=============================================================================
             INŠTALAČNÁ PRÍRUČKA: DECENTRALIZOVANÝ SYSTÉM
=============================================================================

Tento dokument obsahuje presný postup nasadenia a sprevádzkovania 
vyvinutého decentralizovaného systému. 

1. SYSTÉMOVÉ POŽIADAVKY
-----------------------------------------------------------------------------
* Operačný systém: Linux (otestované špecificky pre Ubuntu 24.04 LTS)
* Operačná pamäť (RAM): Minimálne 16 GB pre zabezpečenie plynulého a 
  stabilného chodu celej kontajnerovej infraštruktúry.


2. INŠTALÁCIA PREREKVIZÍT
-----------------------------------------------------------------------------
Pred spustením systému je nevyhnutné pripraviť prostredie a nainštalovať 
potrebné nástroje. Spustite nasledujúce príkazy v termináli.

KROK 2.1: Inštalácia technológie Docker
Najskôr pridajte oficiálny repozitár a nainštalujte najnovšiu verziu Dockeru:

sudo apt update
sudo apt install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
Types: deb
URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
Components: stable
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/docker.asc
EOF

sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

KROK 2.2: Nastavenie oprávnení a spustenie Dockeru
Pridajte svojho používateľa do skupiny docker (namiesto <username> zadajte 
svoje prihlasovacie meno) a povoľte službu:

sudo systemctl start docker
sudo usermod -a -G docker <username>
sudo systemctl enable docker

  (POZNÁMKA: Pre uplatnenie oprávnení môže byť nutné odhlásenie a opätovné 
  prihlásenie do operačného systému).

KROK 2.3: Vytvorenie symbolického odkazu pre docker-compose
Keďže nové verzie používajú plugin Docker Compose v2, pre spätnú kompatibilitu 
(a fungovanie starých názvov príkazov) vytvorte tento odkaz:

sudo ln -s /usr/libexec/docker/cli-plugins/docker-compose /usr/local/bin/docker-compose

KROK 2.4: Inštalácia Node.js a jazyka Go
Pre kompiláciu a beh aplikačných vrstiev sú potrebné nástroje Node.js a Golang:

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs golang-go


KROK 2.5: Konfigurácia prostredia (.env súbory)
Pred samotným spustením je nutné vytvoriť .env konfiguračné súbory. V projekte sa nachádzajú vzorové súbory (.env.example), ktoré je potrebné skopírovať a správne naplniť:

1. Blockchain sieť:
   Skopírujte súbor blockchain-network/prod-network/.env.example do blockchain-network/prod-network/.env
   (Tento súbor zvyčajne nevyžaduje zmeny, definuje len sieťové porty a verzie pre Fabric).
   Príkaz: cp ./blockchain-network/prod-network/.env.example ./blockchain-network/prod-network/.env

2. Backend:
   Skopírujte súbor backend/.env.example do backend/.env
   Príkaz: cp ./backend/.env.example ./backend/.env
   Tento súbor je nutné správne naplniť:
   - JWT_SECRET: Bezpečný tajný kľúč pre generovanie tokenov aplikácie (vložte dlhý náhodný reťazec).
   
   Následne je nutné vytvoriť si účet na službe Pinata ktorá slúži ako IPFS úložisko. 
   V administračnom rozhraní prejdeme do sekcie Gateways, skopírujeme celú adresu 
   brány a vložíme ju do konfiguračného súboru .env v priečinku backend k premennej PINATA_GATEWAY. 
   Potom vytvoríme API kľúč, skopírujeme hodnotu JWT Secret a vložíme ju do toho istého .env 
   súboru k premennej PINATA_JWT.


3. SPUSTENIE BACKENDU A BLOCKCHAINOVEJ SIETE
-----------------------------------------------------------------------------
Základná projektová štruktúra už obsahuje potrebné binárne súbory pre sieť 
Hyperledger Fabric (v adresári blockchain-network/bin). Hlavný inicializačný 
skript sa postará o vygenerovanie certifikátov, spustenie 5 peer uzlov, 
vytvorenie kanála, nasadenie smart kontraktu a spustenie databáz a backend 
serverov pre každú organizáciu.

V koreňovom adresári projektu spustite:

chmod +x *.sh
./start-full-system.sh


4. BEŽNÁ SPRÁVA SYSTÉMU (ZASTAVENIE, SPUSTENIE, AKTUALIZÁCIA)
-----------------------------------------------------------------------------
Na rozdiel od `start-full-system.sh`, ktorý všetko zmaže a vytvorí sieť 
úplne odznova (vrátane prvotného nasadenia chaincode-u pomocou `deployCC.sh`), 
na bežnú prácu a vývoj slúžia tieto skripty:

* Zastavenie systému bez straty dát:
  Zastaví všetky bežiace kontajnery (blockchain aj backendy).
  Príkaz: ./down-system.sh

* Opätovné spustenie systému (zachovanie dát):
  Spustí existujúce kontajnery a obnoví ich stav presne tam, kde skončili.
  Príkaz: ./up-system.sh

* Aktualizácia Smart Kontraktu (Chaincode):
  Ak urobíte zmeny v logike v priečinku `chaincode/`, nemusíte reštartovať 
  a mazať celú sieť. Otvorte súbor `update-cc.sh`, zvýšte premenné 
  CC_VERSION a CC_SEQUENCE (napríklad na 0.2 a 2) a spustite ho.
  Príkaz: ./update-cc.sh
  Sieť automaticky prebalí, nainštaluje, schváli a nasadí novú verziu kódu.


5. SPUSTENIE KLIENTSKEJ APLIKÁCIE (FRONTEND)
-----------------------------------------------------------------------------
Prezentačná vrstva je zjednotená do jednej aplikácie, ktorá umožňuje 
dynamický výber organizácie. 

Otvorte nové okno terminálu a spustite:

cd ./frontend
npm install
npm run dev


6. POUŽÍVANIE APLIKÁCIE A TESTOVANIE
-----------------------------------------------------------------------------
* Prístup: Aplikácia beží na adrese http://localhost:3000
* Prihlásenie: Na uvítacej obrazovke si vyberiete konkrétnu infraštruktúru 
  (portál) organizácie, do ktorej sa chcete prihlásiť.
* Tip pre testovanie: Ak chcete sledovať tok liečiv v reálnom čase z pohľadu 
  viacerých aktérov súčasne (napr. Výrobca a Lekáreň), otvorte si ďalšie 
  portály v okne inkognito vo webovom prehliadači. Zabránite tak nežiaducemu 
  prepisovaniu cookies a relácií medzi rôznymi API portami.
