title "MAPOVANIE PRICE OFFER: Názvoslovný nesúlad (BC vs DB)"

package "HYPERLEDGER_FABRIC (Blockchain)" #E8F5E9 {
    class BC_Order << (P,#B3E5FC) PDC >> {
        * String **requestId** (napr. "REQ-123")
        --
        * priceOffers: [
            { price: 10.5 },
            { price: 9.8 }
        ]
    }
}

package "APLIKAČNÁ_DATABÁZA (MySQL)" #FFF3E0 {
    entity "PriceOffer" as DB_Offer {
        * Int id (PK)
        --
        * String **batchID** (FK)
        * Float price
    }
}

' Vizualizácia nesúladu v mapovaní
BC_Order::requestId ---[#red,bold]---> DB_Offer::batchID : "!!! MAPOVANIE !!!\nBC.requestId sa v DB\nukladá do pola batchID"

BC_Order::priceOffers ..> DB_Offer : "Rozbalenie pola na riadky"

note bottom of DB_Offer
  **POZOR:** Hoci sa stĺpec v databáze 
  volá 'batchID', jeho obsahom je 
  'requestId' z blockchainu.
end note
